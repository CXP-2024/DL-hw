## 1. 项目概览

这个项目要求你在 MNIST 手写数字数据集上实现两种生成模型：
- **能量模型 (EBM)**：用于图像修复（inpainting）—— 恢复被损坏的图像
- **条件 GAN**：用于根据指定数字类别生成新的手写数字图像

### MNIST 数据集
MNIST 是一个包含 70,000 张手写数字（0-9）的灰度图像数据集：
- 训练集：60,000 张
- 测试集：10,000 张
- 图像大小：28×28 像素
- 像素值范围：0（黑色）到 1（白色）

想象一张 28×28 的方格纸，每个格子的颜色深浅代表一个数字。

## 2. 前置知识：什么是生成模型？

### 判别模型 vs 生成模型

**判别模型**（你可能已经熟悉的）：给一张图片，判断它是什么数字。输入→标签。

**生成模型**（本项目的重点）：学习数据的分布，能够**创造**新的、看起来真实的图片。

打个比方：
- 判别模型 = 美术鉴赏师（能判断一幅画好不好）
- 生成模型 = 画家（能画出新的画）

### 为什么需要生成模型？
- 数据增强（生成更多训练数据）
- 图像修复（修复损坏的照片）
- 创意生成（AI 画画）
- 异常检测（判断数据是否"正常"）

## 3. 能量模型 (EBM) 详解

### 3.1 什么是能量模型？

**核心思想**：给每个输入分配一个"能量值"。
- **真实数据** → **低能量**（更稳定，更自然）
- **假数据/噪声** → **高能量**（不自然）

就像物理学中的能量最小原理：一个球会自然滚到最低点。我们训练一个函数 E(x)，让真实图片的能量低，假图片的能量高。

```
E(真实数字图片) = -5  (低能量 = 好)
E(随机噪声图片) = +5  (高能量 = 差)
```

### 3.2 模型架构：MLP（多层感知机）

我们的 EBM 是一个简单的 MLP：

```
输入: 28×28=784 个像素值
  → 展平为 784 维向量
  → 全连接层 784→256 + SiLU 激活
  → 全连接层 256→256 + SiLU 激活
  → 全连接层 256→256 + SiLU 激活
  → 全连接层 256→1
输出: 1 个标量（能量值）
```

**SiLU 激活函数**：`SiLU(x) = x * sigmoid(x)`，比 ReLU 更平滑，有助于梯度计算。

### 3.3 训练方法：去噪分数匹配 (Denoising Score Matching, DSM)

DSM 的核心思想非常优雅：

1. 取一张干净图片 `x`
2. 加噪声得到 `x̃ = x + σε`（ε 是随机噪声）
3. 训练模型的**梯度** `∇E(x̃)` 指向从噪声→干净的方向

具体来说，训练目标是：
```
损失 = ||∇E(x̃) - (x̃ - x)/σ²||²
```

这意味着：模型的梯度应该等于 `(噪声图片 - 干净图片)/σ²`，即指向"从噪声恢复到干净"的方向。

**在本项目中的特殊处理**：

由于评估时使用的是"隔行损坏"模式（偶数行保留，奇数行加噪声），我们**在训练时也使用完全相同的损坏模式**。这样模型学到的梯度就能直接用于修复。

### 3.4 图像修复 (Inpainting) 原理

修复过程使用**朗之万动力学 (Langevin Dynamics)**：

```python
# 从损坏的图像出发
x = corrupted_image

# 重复多次：
for i in range(80):
    # 1. 计算能量梯度
    grad = ∇E(x)

    # 2. 沿梯度方向移动（降低能量）
    x = x - step_size * grad

    # 3. 保持已知像素不变
    x[已知像素] = corrupted_image[已知像素]

    # 4. 限制像素值在 [0, 1] 范围内
    x = clamp(x, 0, 1)
```

**关键参数**：
- `step_size = 0.015`：每步移动的幅度
- `n_steps = 80`：总步数

这些参数需要与训练时的梯度尺度匹配，否则步子太大会发散，太小会收敛不了。

### 3.5 损坏模式详解

评估脚本的损坏方式：
```
原始图像（28行）：
行0: ████████ (偶数行 → 保留)
行1: ▓▓▓▓▓▓▓▓ (奇数行 → 替换为 0.3×随机噪声)
行2: ████████ (偶数行 → 保留)
行3: ▓▓▓▓▓▓▓▓ (奇数行 → 替换为噪声)
...
```

模型需要根据保留的偶数行，推测奇数行的内容。

## 4. 生成对抗网络 (GAN) 详解

### 4.1 什么是 GAN？

GAN 由两个网络对抗训练：

**生成器 (Generator, G)**：接收随机噪声 z，生成假图片
**判别器 (Discriminator, D)**：判断图片是真是假

```
        随机噪声 z
            ↓
    ┌───────────────┐
    │   生成器 G     │ ← 想要骗过 D
    └───────┬───────┘
            ↓
        假图片 G(z)
            ↓
    ┌───────────────┐
    │   判别器 D     │ ← 想要分辨真假
    └───────┬───────┘
            ↓
        真/假 判断
```

打个比方：
- G = 造假者（试图画出逼真的假画）
- D = 鉴定师（试图识破假画）
- 两者互相竞争，最终 G 学会画出极其逼真的画

### 4.2 条件 GAN (Conditional GAN)

普通 GAN 随机生成图片，条件 GAN 可以**指定生成什么**。

```
输入: 噪声 z + 标签 "3"
输出: 一张手写数字 3 的图片
```

实现方式：通过**标签嵌入 (Label Embedding)**
- 将数字 0-9 映射为 50 维向量
- 与噪声向量拼接后输入生成器
- 判别器也接收标签信息，判断"这张图片是否像指定的数字"

### 4.3 生成器架构 (DCGAN 风格)

```
输入: z(100维) + label_embedding(50维) = 150维
  → 全连接层 150 → 256×7×7 = 12544
  → 重塑为 (256, 7, 7) 的特征图
  → BatchNorm + ReLU
  → 转置卷积 256→128, 步长2 → (128, 14, 14)  ← 上采样
  → BatchNorm + ReLU
  → 转置卷积 128→1, 步长2 → (1, 28, 28)       ← 上采样
  → Tanh → 输出范围 [-1, 1]
输出: 28×28 的图像
```

**转置卷积 (ConvTranspose2d)**：卷积的"反向"操作，用于上采样（放大特征图）。

### 4.4 判别器架构

```
输入: 图像(1, 28, 28) + label_map(1, 28, 28) = (2, 28, 28)
  → 卷积 2→64, 步长2 → (64, 14, 14) + LeakyReLU  ← 下采样
  → 卷积 64→128, 步长2 → (128, 7, 7) + LeakyReLU  ← 下采样
  → 卷积 128→1 → (1, 1, 1)
输出: 1 个标量（真/假分数）
```

**谱归一化 (Spectral Normalization)**：限制判别器的 Lipschitz 常数，防止判别器太强导致训练不稳定。

**label_map**：将标签嵌入为 28×28 的"图像"，与原图在通道维度拼接。这样判别器在每个空间位置都能看到标签信息。

### 4.5 训练过程：Hinge Loss

传统 GAN 使用 BCE 损失，我们使用更稳定的 **Hinge Loss**：

```python
# 判别器损失
D_loss = max(0, 1 - D(real)) + max(0, 1 + D(fake))

# 生成器损失
G_loss = -D(fake)
```

**为什么 Hinge Loss 更好？**
- 一旦判别器对某个样本足够自信（分数超过 1），就不再继续"惩罚"
- 这防止判别器变得太强，维持 D 和 G 的平衡

### 4.6 关键训练技巧

1. **谱归一化**：稳定判别器，防止梯度爆炸
2. **Adam 优化器** (β₁=0.5, β₂=0.999)：GAN 训练的标准选择
3. **学习率 2e-4**：DCGAN 推荐的学习率
4. **训练 80 个 epoch**：足够模型收敛

## 5. 代码实现逐行讲解

### 5.1 EBM 模型 (`modules/ebm.py`)

```python
class CustomEBMModel(nn.Module):
    def __init__(self):
        super().__init__()
        # 4层 MLP: 784→256→256→256→1
        self.net = nn.Sequential(
            nn.Flatten(),           # (batch, 1, 28, 28) → (batch, 784)
            nn.Linear(784, 256),    # 第1层
            nn.SiLU(),              # 激活函数
            nn.Linear(256, 256),    # 第2层
            nn.SiLU(),
            nn.Linear(256, 256),    # 第3层
            nn.SiLU(),
            nn.Linear(256, 1),      # 输出层：1个能量值
        )

    def forward(self, x):
        return self.net(x)          # 返回能量值 (batch, 1)

    def inpaint(self, corrupted_images, mask):
        x = corrupted_images.clone().detach()

        for _ in range(80):                        # 80步朗之万动力学
            x.requires_grad_(True)                 # 开启梯度追踪
            energy = self.forward(x).sum()         # 计算总能量
            energy.backward()                       # 反向传播得到梯度
            grad = x.grad.detach()                  # 获取 ∂E/∂x

            with torch.no_grad():
                x = x - 0.015 * grad               # 沿负梯度移动
                x = torch.where(mask, corrupted_images, x)  # 保持已知像素
                x = x.clamp(0, 1)                   # 限制范围
                x = x.detach()                      # 停止梯度追踪

        return x
```

### 5.2 EBM 训练 (`train_ebm.py`)

```python
def train(model, dataset):
    # 超参数
    batch_size = 128
    lr = 1e-3
    n_epochs = 100

    # 两种训练信号：
    # 1. 行损坏 DSM —— 直接训练修复能力
    # 2. 均匀噪声 DSM —— 泛化能力

    for epoch in range(n_epochs):
        for images in dataloader:
            # === 行损坏 DSM ===
            corrupted, mask = corrupt_images(images)    # 隔行加噪
            corrupted.requires_grad_(True)
            energy = model(corrupted).sum()
            grad = autograd.grad(energy, corrupted, create_graph=True)[0]

            target = corrupted - images                  # 目标梯度方向
            dsm_loss = ((grad - target) * corrupted_mask).square().mean()

            # === 均匀噪声 DSM ===
            sigma = random(0.2, 0.5)
            x_noisy = images + sigma * randn()
            grad2 = autograd.grad(model(x_noisy).sum(), x_noisy, create_graph=True)[0]
            target2 = (x_noisy - images) / sigma²
            dsm_loss2 = ((grad2 - target2)²).mean()

            loss = dsm_loss + 0.1 * dsm_loss2            # 综合损失
            loss.backward()
            optimizer.step()
```

### 5.3 GAN 生成器 (`modules/gan.py`)

```python
class CustomGANGenerator(nn.Module):
    def __init__(self):
        self.latent_dim = 100                           # 噪声向量维度
        self.label_emb = nn.Embedding(10, 50)           # 10个类别→50维嵌入
        self.proj = nn.Linear(150, 256*7*7)             # 投影到特征图
        self.main = nn.Sequential(
            nn.BatchNorm2d(256), nn.ReLU(True),
            nn.ConvTranspose2d(256, 128, 4, 2, 1),     # 7×7 → 14×14
            nn.BatchNorm2d(128), nn.ReLU(True),
            nn.ConvTranspose2d(128, 1, 4, 2, 1),       # 14×14 → 28×28
            nn.Tanh(),                                   # 输出 [-1, 1]
        )

    def generate(self, labels):
        z = torch.randn(len(labels), 100, device=labels.device)
        images = self.forward(z, labels)
        return (images + 1) / 2                         # [-1,1] → [0,1]
```

### 5.4 GAN 训练 (`train_gan.py`)

```python
def train(generator, discriminator, dataset):
    for epoch in range(80):
        for real_images, labels in dataloader:
            real_images = real_images * 2 - 1           # [0,1] → [-1,1]

            # --- 训练判别器 ---
            d_real = discriminator(real_images, labels)
            z = torch.randn(batch_size, 100)
            fake = generator(z, labels).detach()
            d_fake = discriminator(fake, labels)

            # Hinge loss
            d_loss = relu(1-d_real).mean() + relu(1+d_fake).mean()
            d_loss.backward()
            optimizer_d.step()

            # --- 训练生成器 ---
            z = torch.randn(batch_size, 100)
            fake = generator(z, labels)
            d_fake = discriminator(fake, labels)
            g_loss = -d_fake.mean()                     # 让 D 认为假图是真的
            g_loss.backward()
            optimizer_g.step()
```

## 6. 训练过程和超参数选择

### 6.1 EBM 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| batch_size | 128 | 每批样本数 |
| learning_rate | 1e-3 | 较大的学习率加速 DSM 收敛 |
| epochs | 100 | 训练轮数 |
| 噪声范围σ | [0.2, 0.6] | 多尺度噪声训练 |
| 修复 step_size | 0.015 | 朗之万动力学步长 |
| 修复 n_steps | 80 | 朗之万动力学步数 |
| 优化器 | Adam | 学习率调度：CosineAnnealing |

### 6.2 GAN 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| batch_size | 128 | 每批样本数 |
| lr_g | 2e-4 | 生成器学习率 |
| lr_d | 2e-4 | 判别器学习率 |
| epochs | 80 | 训练轮数 |
| latent_dim | 100 | 噪声向量维度 |
| 损失函数 | Hinge Loss | 比 BCE 更稳定 |
| 判别器正则化 | Spectral Norm | 防止判别器过强 |
| 优化器 | Adam(β₁=0.5, β₂=0.999) | GAN 标准配置 |

### 6.3 为什么选择这些超参数？

**EBM**:
- `lr=1e-3`：DSM 训练不如 CD 那样不稳定，可以用较大学习率
- `step_size=0.015`：通过网格搜索找到的最优值，太大会发散，太小收敛慢
- `n_steps=80`：足够收敛，同时保持评估速度

**GAN**:
- Hinge Loss + Spectral Norm 是经验证的稳定组合
- `β₁=0.5`：GAN 常用设置，低于默认的 0.9，减少动量带来的震荡
- 80 epochs：给模型足够时间学习，但不过度训练

## 8. 常见问题和调试技巧

### Q: EBM 训练时损失接近 0 是正常的吗？
A: 如果使用纯对比散度(CD)训练，损失接近 0 说明模型没有学到有意义的能量差异。这通常意味着 L2 正则化系数太大。使用 DSM 训练可以避免这个问题。

### Q: GAN 出现模式崩溃(mode collapse)怎么办？
A: 模式崩溃是指生成器只生成少数几种图片。解决方法：
1. 使用谱归一化(Spectral Normalization)限制判别器
2. 使用 Hinge Loss 代替 BCE Loss
3. 确保判别器和生成器的训练速度平衡

### Q: 为什么我的 GAN D_loss 很低但 G_loss 很高？
A: 这说明判别器太强了，生成器追不上。可以：
- 降低判别器学习率
- 给判别器添加 Spectral Normalization
- 每训练 1 次判别器，训练 2 次生成器

### Q: EBM 的修复步长(step_size)怎么选？
A: 步长需要与训练时的梯度尺度匹配。使用网格搜索在 [0.005, 0.1] 范围内找到最优值。建议先在少量数据上测试。

### Q: 为什么要用 Tanh 而不是 Sigmoid 作为 GAN 生成器的最后一层？
A: Tanh 输出范围 [-1, 1]，梯度比 Sigmoid 更稳定。训练时将真实图片也缩放到 [-1, 1]，生成时再转回 [0, 1]。

### Q: 什么是 Spectral Normalization？
A: 谱归一化通过将权重矩阵的每个元素除以其最大奇异值来归一化。这确保了网络的每一层都满足 Lipschitz 连续性约束，从而防止判别器的梯度变得过大或过小。

## 附录：核心数学公式

### 朗之万动力学
$$x_{t+1} = x_t - \frac{\epsilon}{2} \nabla_x E(x_t) + \sqrt{\epsilon} \cdot \mathcal{N}(0, I)$$

### 对比散度 (CD) 损失
$$L_{CD} = E_\theta(x^+) - E_\theta(x^-) + \alpha(E_\theta(x^+)^2 + E_\theta(x^-)^2)$$

### 去噪分数匹配 (DSM) 损失
$$L_{DSM} = \mathbb{E}\left[\left\|\nabla_{\tilde{x}} E_\theta(\tilde{x}) - \frac{\tilde{x} - x}{\sigma^2}\right\|^2\right]$$

### GAN Hinge 损失
$$L_D = \mathbb{E}[\max(0, 1-D(x_{real}))] + \mathbb{E}[\max(0, 1+D(G(z)))]$$
$$L_G = -\mathbb{E}[D(G(z))]$$
